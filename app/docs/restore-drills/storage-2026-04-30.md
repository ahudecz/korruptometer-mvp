# Storage restore drill — 2026-04-30

Launch-gate T127. Verifies sibling-region snapshot recovery for the
`submissions` Supabase Storage bucket (FR-057, SC-020).

## Status

`PENDING` — must be exercised on staging before spec-Phase-2 ships.

## Procedure

1. Pick a random object from `submissions/` in the latest sibling-region
   snapshot. Record its `key` and `Content-Length`.
2. Restore it to a staging bucket via `supabase storage cp`:
   ```sh
   supabase storage cp --backup s3://supabase-snap-eu-west-1/<key> staging-bucket/<key>
   ```
3. Verify byte-equal restore: `sha256sum` of the restored object must match
   the snapshot manifest's recorded hash.
4. Run the orphan-scan dry-run against the staging bucket and confirm the
   restored object is correctly recognised (`SubmissionAttachment.storageKey`
   matches → not orphaned).
